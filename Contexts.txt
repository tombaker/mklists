We have been speaking of the "context layers":

- ConfigContext (derived from config files)
- StructuralContext (derived from filesystem structure)
- ExecutionContext (fully resolved, ready to execute)
- plus ExecutionPass (per-pass execution unit)

as if processing should start with deriving ConfigContext then continue on to StructuralContext. 

However, it looks to me like resolving ConfigContext depends on first resolving StructuralContext, because 


---------------------------------------------------------
ConfigContext - was: MklistsConfig
- config.py (still)

@dataclass(slots=True, frozen=True)
class BackupConfig:
    backup_enabled: bool
    backup_rootdir: Path
    backup_depth: int


@dataclass(slots=True, frozen=True)
class RoutingConfig:
    routing_enabled: bool
    routing_dict: dict[str, Path]


@dataclass(slots=True, frozen=True)
class UrlifyConfig:
    urlify_enabled: bool
    urlify_dir: Path


@dataclass(slots=True, frozen=True)
class ConfigContext:
    """Normalized configuration derived from defaults + user config."""

    verbose: bool
    backup: BackupConfig
    routing: RoutingConfig
    urlify: UrlifyConfig
