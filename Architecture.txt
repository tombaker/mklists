Architectural Invariants
1.  Structural Resolution Layer
    Derive fully defined structural Execution Universe from filesystem.

    Input from CLI or CWD:

        startdir: Path

    Output resolved execution context for a single run:

        RunContext
            config_rootdir: Path
            repo_configfile: Path | None
            repo_rulefile: Path | None
            datadir_contexts: list[DatadirContext]

            DatadirContext
                datadir: Path
                configfile_used: Path | None
                rules: list[Rule]

2.  Configuration Resolution Layer
    Load config for processing one or more datadirs and normalize into typed config.

    Input:

        RunContext.config_rootdir, which includes:
        - DatadirContext.configfile_used

    Output configuration to be applied in processing one or more datadirs:

        MklistsConfig
            verbose: bool
            backup: BackupConfig
            routing: RoutingConfig
            safety: SafetyConfig
            urlify: UrlifyConfig

3.  Planning Layer
    Determine datadirs to process, passes, routing dictionary, backup/html directories.

    Input:

        RunContext
        MklistsConfig

    Output:

        RunPlan:
            datadir_contexts: list[DatadirContext]
            pass_plans: list[PassPlan]
            routing_dict: dict
            htmldir: Path

            PassPlan:
                backupdir: Path | None

            DatadirContext
                datadir: Path
                configfile_used: Path | None
                rules: list[Rule]

    Invariants:
    - Planning is deterministic.
    - Planning causes no side effects (eg, filesystem mutations).
    - If only one datadir: one pass. 
    - If multiple datadirs and routing enabled: two passes.
    - Backup directory paths are computed but not yet created.

    Planning guarantees:
    - All paths are absolute.
    - All passes are enumerated.
    - No structural ambiguity remains.

4.  Execution layer ("run")

    Input:

        RunPlan

    Output:
        
        None, after executing passes.

    Note:
        If backups are enabled, per-pass backupdir is created before any read-write.

5.  Backup system invariants

    - Backup directories under `config_rootdir / backupdir` unless explicitly absolute.
    - Pruning enforces backup depth after each new backup creation.

6.  Global architectural constraints

    Filesystem
       ↓
    RunContext (structure)
       ↓
    MklistsConfig (configuration)
       ↓
    RunPlan (planning)
       ↓
    Execution (mutation)

7.  Mental Model
    - Structure defines universe.
    - Config defines policy.
    - Plan defines sequence.
    - Execution performs mutation.
    - Each layer is strictly narrower than the one above it.
